2 Architecture technique

Flux de données principaux

Flux 1 : Création de recette

Frontend Form→ Zod validation→ POST /api/v1/recettes→ Pydantic RecetteCreate→ ServiceRecettes.create_complete()→ Normalisation (mojibake, catégories)→ Find/Create Ingrédients→ Persist + Event "recette.creee"

Flux 2 : Génération liste de courses

Planning validé→ POST generer-courses-depuis-planning→ Query repas + recettes→ Agrégation ingrédients→ Soustraction inventaire→ Suggestion rayons + magasins→ ListeCourses créée

Flux 3 : Session Batch Cooking

Sélection recettes planning→ POST generer-depuis-planning→ Calcul parallélisation robots→ Génération étapes IA→ Session (planifiée→en cours→terminée)→ Préparations stockées (frigo/congel)→ Consommation via planning

Stack technique

Backend

  • FastAPI + SQLAlchemy 2.0 ORM
  • Pydantic v2 (validation/sérialisation)
  • Mistral AI (suggestions, adaptation)
  • Services avec mixins + @service_factory
  • Cache multi-niveaux (@avec_cache)
  • Event bus pub/sub
  • Rate limiting (60/10 req/min)

Frontend

  • Next.js 16 (App Router, Turbopack)
  • TypeScript 5 + Tailwind CSS v4
  • TanStack Query v5 (data fetching)
  • Zustand 5 (state)
  • react-hook-form 7 + Zod (forms)
  • dnd-kit (Drag-and-Drop)
  • framer-motion (animations)

Infrastructure

  • Supabase PostgreSQL (28 tables cuisine)
  • RLS policies par utilisateur
  • Migrations SQL-file
  • WebSocket (courses temps réel)
  • OpenFoodFacts API (enrichissement)
  • Extension Chrome (Carrefour Drive)

8 Bugs et problèmes identifiés

Erreurs de compilation (Pyright)

52 erreurs de type dans batch_cooking.py

Problème principal : Les dictionnaires REPONSES_* (ex: REPONSES_CRUD_CREATION) ont un type dict[int, dict[str, Any]] mais FastAPI attend Dict[int | str, Dict[str, Any]]. Pyright refuse car int n'est pas assignable à int | str en mode invariant.

Impact : Non bloquant à l'exécution (Python runtime les accepte), mais pollue l'analyse statique.

Fix : Typer les constantes comme dict[int | str, dict[str, Any]] dans schemas/errors.py.

~20 erreurs Unknown type dans la génération d'étapes batch

Dicts non typés (dict = {}) dans la fonction generer_etapes_session() propagent des types Unknown sur toutes les opérations qui suivent (get, setdefault, sorted).

Fix : Typer explicitement : planning_data: dict[str, dict[str, dict]] = {}
